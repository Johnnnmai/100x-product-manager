# shape-idea

Use this when the starting point is a rough idea, a founder instinct, or a weakly formed opportunity.

## Input

- rough idea
- notes or fragments
- optional constraints
- optional category or market

## Output

- problem statement
- target user hypothesis
- wedge recommendation
- scope-down recommendation
- top risks
- first success metric draft
- next recommended command

## Default Workflow

1. Explore multiple interpretations with `idea-explorer`
2. Pick the strongest initial wedge with `opportunity-wedge-selector`
3. Reduce ambition to a shippable first move with `idea-scope-down-architect`
4. Strengthen the framing with `problem-framing-canvas`
5. Draft the first success condition with `success-metric-spec-writer`

## Operating Rules

- Default to autonomous first-pass exploration.
- Generate 2-3 viable directions before recommending one.
- Do not ask clarifying questions unless the missing context blocks all useful first-pass work.
- Make assumptions explicit.
