# make-content

Use this when internal product work should become reusable external distribution.

## Input

- PRD
- experiment result
- decision memo
- launch note
- case study fragments

## Output

- XHS note draft
- SEO brief
- GEO entity note
- short founder distribution summary

## Default Workflow

1. Extract the proof-of-work angle
2. Convert the work into a platform-friendly artifact
3. Preserve the decision signal, not just the conclusion
