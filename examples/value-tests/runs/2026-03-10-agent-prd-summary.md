# Agent PRD Summary

## Result

`agent-prd` beat the plain prompt on quality and time to acceptable draft.

## Why 100X PM Won

- it required a spec that works for humans and agents
- it forced dependencies, constraints, and acceptance checks into the artifact
- it added fallback behavior instead of assuming the model always works

## Why Baseline Lost

- it looked like a lightweight feature brief, not a real execution spec
- it had goals and features, but not task contracts
- it had risks, but no rule for what happens when those risks materialize

## Caveat

This is still a pilot score. The stronger result here comes from structure quality, not external validation.
