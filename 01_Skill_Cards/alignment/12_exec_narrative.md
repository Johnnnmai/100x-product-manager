# Skill Card: Exec Narrative (One Pager)

## Definition

Create a compelling one-page narrative thataligns executives on the "why, what, and what's next" of an initiative.

---

## When to Use

- Before executive presentations
- For project funding requests
- When alignment is needed across teams
- For any significant initiative

---

## Outputs (Artifacts)

| Artifact | Description | Required? |
|----------|-------------|------------|
| Exec One-Pager | Single-page summary | Yes |
| Problem Statement | Business problem | Yes |
| Solution Summary | What and why | Yes |
| Impact Quantification | Expected results | Yes |
| FAQ / Objections | Anticipated questions | Yes |
| Ask | What you need from execs | Yes |

---

## Pass Criteria

- [ ] Fits on one page
- [ ] Problem clearly stated with data
- [ ] Solution explained simply
- [ ] Impact quantified
- [ ] FAQ covers 10+ objections
- [ ] Clear "ask" defined

---

## Red Flags

- ⚠️ More than one page
- ⚠️ No data to support problem
- ⚠️ Solution too technical
- ⚠️ Impact not quantified
- ⚠️ No clear ask

---

## Example

**Scenario:** New feature investment request

**One-Pager:**
- Problem: 40% of users drop at onboarding step 3
- Solution: Simplify to 2-step process
- Impact: +15% activation = $2M/year revenue
- Ask: $50K engineering investment
- FAQ: "Why not X?", "What about Y?", etc.

---

## Context Adapter

### Startup Version
- Minimum artifacts: One-Pager + Problem + Ask
- Timeline: 1 day
- Focus: Survival and growth metrics

### Big Tech Version
- Required artifacts: All 6 artifacts
- Timeline: 1 week
- Focus: Org alignment + risk + resource justification
