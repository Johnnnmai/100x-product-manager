# Skill Card: Stakeholder Map & Decision Owner

## Definition

Identify all stakeholders, their interests, influence, and who owns each key decision.

---

## When to Use

- At project start
- When stakeholder conflicts arise
- Before any significant decision
- For cross-functional projects

---

## Outputs (Artifacts)

| Artifact | Description | Required? |
|----------|-------------|------------|
| Stakeholder Map | All parties involved | Yes |
| Interest/Influence Matrix | Who cares about what | Yes |
| Decision Owner | Who owns each decision | Yes |
| Decision Log | Past decisions recorded | Yes |
| Escalation Path | How to resolve blocks | Conditional |

---

## Pass Criteria

- [ ] All stakeholders identified
- [ ] Each key decision has owner
- [ ] Decision rights documented
- [ ] Escalation path defined
- [ ] Conflicts identified and managed

---

## Red Flags

- ⚠️ Stakeholder list incomplete
- ⚠️ No clear decision owner
- ⚠️ Stakeholder conflicts not addressed
- ⚠️ No escalation path

---

## Example

**Scenario:** New pricing model

**Stakeholder Map:**
- VP Product (final decision)
- VP Eng (technical feasibility)
- VP Sales (customer impact)
- Finance (pricing approval)
- Customer Success (support)

**Decision Owners:**
- Pricing: VP Product + Finance
- Technical approach: VP Eng
- Timeline: Product Manager

---

## Context Adapter

### Startup Version
- Minimum artifacts: Stakeholder Map + Decision Owner
- Timeline: 1 day
- Typically: Founder as final decision

### Big Tech Version
- Required artifacts: All 5 artifacts
- Timeline: 1 week
- Multiple layers: DRIs, reviewers, approvers
