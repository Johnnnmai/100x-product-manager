# Skill Card: MVP Cutline & Non-goals

## Definition

Define the minimum set of features required to achieve the outcome, and explicitly state what is NOT in scope.

---

## When to Use

- When scope keeps expanding
- Before writing PRD or technical spec
- When multiple stakeholders have different expectations
- For any new product or major feature

---

## Outputs (Artifacts)

| Artifact | Description | Required? |
|----------|-------------|------------|
| PRD One-Pager | Single-page product definition | Yes |
| MVP Cutline | Minimum features to ship | Yes |
| Non-goals | Explicit out-of-scope items | Yes |
| Trade-off Matrix | What we're trading off | Yes |
| User Story Mapping | Feature prioritization | Conditional |

---

## Pass Criteria

- [ ] MVP can achieve the stated outcome (not just "feature complete")
- [ ] Each requirement answers: "If we don't do this, does it break the outcome?"
- [ ] Non-goals are explicitly documented
- [ ] Trade-offs are acknowledged (speed vs quality, scope vs time)
- [ ] Stakeholders have signed off on cutline

---

## Red Flags

- ⚠️ MVP keeps expanding
- ⚠️ No explicit non-goals
- ⚠️ Can't articulate why each feature is essential
- ⚠️ Stakeholders disagree on scope
- ⚠️ "Nice to have" mixed with "must have"

---

## Example

**Scenario:** Building a file sharing product

**MVP Cutline:**
- Upload file (must)
- Download file (must)
- Share via link (must)
- Basic authentication (must)

**Non-goals:**
- File preview (Phase 2)
- Version history (Phase 2)
- Team workspaces (Phase 3)
- Mobile app (Phase 3)

**Result:** Shipped in 4 weeks, validated product-market fit

---

## Context Adapter

### Startup Version
- Minimum artifacts: MVP Cutline + Non-goals
- Timeline: 1 day
- Cutline based on: Speed to validate

### Big Tech Version
- Required artifacts: All 5 artifacts
- Timeline: 1 week
- Cutline based on: Scale + long-term maintenance
