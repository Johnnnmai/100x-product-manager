# Playbook: Startup MVP 0 to 1

## When to Use

- Building a new product from scratch
- Validating product-market fit
- Limited resources and time

## Duration

**2-4 weeks**

## Required Skills

| Order | Skill Card | Artifacts | Duration |
|-------|------------|-----------|----------|
| 1 | 01_problem_framing | Problem Statement, Outcome Spec | 1-2 days |
| 2 | 05_mvp_cutline | MVP Cutline, Non-goals | 1 day |
| 3 | 07_milestone_plan | Milestone Plan | 1 day |
| 4 | 03_metrics_tree | Metrics Tree | 1 day |
| 5 | 09_rollout_rollback | Rollout Plan | 1 day |

## Optional Skills

| Skill Card | When to Use |
|------------|-------------|
| 02_service_blueprint | If user journey is complex |
| 04_experiment_design | If you have enough users to test |

## Execution Flow

```
Week 1:
├── Day 1-2: Problem Framing
│   ├── Interview 5-10 users
│   ├── Define outcome spec
│   └── Validate problem
│
├── Day 3-4: MVP Cutline
│   ├── Define minimum features
│   ├── Explicit non-goals
│   └── Get alignment
│
└── Day 5: Metrics Tree
    ├── Define success metrics
    └── Plan instrumentation

Week 2:
├── Day 1-2: Milestone Plan
│   ├── Weekly milestones
│   └── Critical path
│
└── Day 3-5: Build
    ├── Sprint execution
    └── Daily standups

Week 3:
├── Day 1-3: Build & Test
│   ├── Internal testing
│   └── Bug fixes
│
└── Day 4-5: Rollout Prep
    ├── Rollout plan
    └── Simple launch

Week 4:
└── Launch + Measure
    ├── 1% - 10% - 50% rollout
    └── Gather feedback
```

## Success Criteria

- [ ] Problem validated with real users
- [ ] MVP shipped to real users
- [ ] Metrics instrumented
- [ ] Feedback loop established
- [ ] Decision: Pivot / Persevere / Stop

## Key Principles

1. **Speed over perfection**: Ship in 2-4 weeks
2. **Manual fallbacks OK**: Don't over-engineer
3. **Interview users**: Not just internal opinion
4. **Small batches**: Rollout gradually

## Common Mistakes to Avoid

- ⚠️ Building too much in v1
- ⚠️ Waiting for "perfect" product
- ⚠️ Not talking to users
- ⚠️ No metrics to measure success
