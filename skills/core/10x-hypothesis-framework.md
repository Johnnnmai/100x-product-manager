# 10x Hypothesis Framework / 10倍增长假设框架

[English](#english) | [中文](#中文)

---

## English

### When to Use This Skill

Use this when you need to think about **exponential growth** rather than incremental improvements.

### Core Question

**"What change can bring 10x growth?"**

### The Problem with 10%

Most PMs think in terms of 10% improvements:
- "How do we increase conversion by 10%?"
- "How do we grow 10% faster?"

But 10% thinking leads to 10% results. The real question is:

**"What's the lever that could 10x our business?"**

### The Framework

#### Step 1: Identify Current Constraints

What are the bottlenecks limiting your growth?

| Category | Examples |
|----------|----------|
| Acquisition | Traffic, conversion rate |
| Activation | Sign-up rate, onboarding completion |
| Retention | Churn, engagement |
| Revenue | ARPU, frequency |
| Virality | Referral rate, NPS |

#### Step 2: Ask the 10x Question

For each category, ask:
- "What if this was 10x better?"
- "What would 10x look like?"
- "What would we do differently?"

**Examples:**
- 10x more traffic → Viral loop? SEO? Paid acquisition?
- 10x better conversion → Better onboarding? AI personalization?
- 10x retention → Different product? Subscription model?
- 10x ARPU → New pricing? Premium tier? Enterprise?

#### Step 3: Identify the 10x Levers

Rank your opportunities by potential:

| Lever | Current | 10x Potential | Effort | Confidence |
|-------|---------|---------------|--------|------------|
| | | | High/Med/Low | High/Med/Low |

#### Step 4: Design Experiments

For each top lever, design experiments:

**Template:**
- **Hypothesis:** If [lever change], then [metric] will 10x because [reason]
- **Test:** [Minimum test description]
- **Timeline:** [How fast can we test?]
- **Success:** [What proves 10x potential?]

### The 10x Mindset

| 10% Thinking | 10x Thinking |
|--------------|--------------|
| Optimize existing | Redefine the problem |
| Incremental | Exponential |
| Reduce friction | Remove the need |
| Better features | Different model |
| More traffic | Viral loops |
| Lower churn | Delight users |

### Output Template

| Lever | 10x Vision | Key Experiment | Timeline | Priority |
|-------|-----------|----------------|----------|----------|
| | | | | |

### Key Principle

**Think Exponential:**
- Don't ask "How do we grow 10%?"
- Ask "What would 10x growth require?"
- Then work backwards from there

## 中文

### 何时使用此技能

当需要思考**指数级增长**而非渐进式改进时使用。

### 核心问题

**"什么变化能带来10倍增长？"**

### 10%的问题

大多数 PM 以10%的改进来思考：
- "我们如何提高10%转化率？"
- "我们如何增长10%？"

但10%思维导致10%的结果。真正的问题是：

**"什么杠杆能让我们业务增长10倍？"**

### 框架

#### 第一步：识别当前瓶颈

什么限制了你的增长？

| 类别 | 例子 |
|------|------|
| 获取 | 流量、转化率 |
| 激活 | 注册率、入驻完成率 |
| 留存 | 流失、参与度 |
| 收入 | ARPU、频率 |
| 病毒式 | 推荐率、NPS |

#### 第二步：问10倍问题

对于每个类别，问：
- "如果这个好10倍会怎样？"
- "10倍会是什么样子？"
- "我们会有什么不同的做法？"

**例子：**
- 10倍流量 → 病毒式？SEO？付费获取？
- 10倍转化 → 更好的入驻？AI个性化？
- 10倍留存 → 不同产品？订阅模式？
- 10倍 ARPU → 新定价？高级版？企业版？

#### 第三步：识别10倍杠杆

按潜力排序机会：

| 杠杆 | 当前 | 10倍潜力 | 投入 | 信心 |
|------|------|----------|------|------|
| | | | 高/中/低 | 高/中/低 |

#### 第四步：设计实验

对于每个顶级杠杆，设计实验：

**模板：**
- **假设：** 如果[杠杆变化]，[指标]将10倍[原因]
- **测试：** [最小测试描述]
- **时间线：** [我们能多快测试？]
- **成功：** [什么证明10倍潜力？]

### 10倍心态

| 10%思维 | 10倍思维 |
|---------|----------|
| 优化现有 | 重新定义问题 |
| 渐进式 | 指数式 |
| 减少摩擦 | 消除需求 |
| 更好功能 | 不同模式 |
| 更多流量 | 病毒式循环 |
| 降低流失 | 取悦用户 |

### 输出模板

| 杠杆 | 10倍愿景 | 关键实验 | 时间线 | 优先级 |
|------|----------|----------|--------|--------|
| | | | | |

### 关键原则

**指数思维：**
- 不要问"我们如何增长10%？"
-问"10倍增长需要什么？"
- 然后从那里倒推
