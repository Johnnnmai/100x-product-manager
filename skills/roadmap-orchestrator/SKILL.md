# What you get

- roadmap themes
- milestone plan

# Common failure

Roadmaps list projects without making sequence, dependencies, and ownership legible.

# Best for

- teams turning direction into milestone plans
- PMs splitting a roadmap into workable streams

# 3-5 key questions

1. What objective anchors the roadmap?
2. Which milestones matter most?
3. What dependencies shape sequence?
4. What must be true before the next stage?

# Working logic

1. Turn direction into a small set of themes.
2. Sequence work by dependency and learning value.
3. Split the work into milestone-sized chunks.

# Output format

- roadmap themes
- milestones
- dependency notes
- workstream split

# Review checklist

- Is the roadmap sequenced, not just listed?
- Are dependencies explicit?
- Are milestones concrete enough to manage?
