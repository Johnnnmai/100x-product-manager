# What you get

- 2-3 viable idea frames
- target user hypotheses
- wedge options

# Common failure

Teams treat the first phrasing of an idea as the best framing. This skill broadens the search before narrowing the scope.

# Best for

- PMs and founders starting from a rough idea
- Teams that need useful first-pass exploration without waiting on interviews

# 3-5 key questions

1. What raw idea or instinct exists today?
2. What user problem might sit underneath it?
3. What outcome would make this worth doing?
4. What constraints already matter?

# Working logic

1. Translate the rough idea into several plausible problem frames.
2. Identify who each frame serves first.
3. Compare which frame creates the strongest wedge.
4. Recommend one starting direction and keep alternatives visible.

# Output format

- 2-3 framed directions
- likely user for each
- strongest wedge recommendation
- risks and assumptions

# Review checklist

- Are the options materially different, not cosmetic rewrites?
- Is the user hypothesis explicit?
- Is one direction recommended?
