# What you get

- wedge recommendation
- why this wedge wins first
- wedge risks

# Common failure

Teams pick a wedge based on prestige or market size instead of learning speed and execution feasibility.

# Best for

- early-stage products choosing a first market entry point
- PMs narrowing a broad opportunity

# 3-5 key questions

1. Which user segment feels most acute?
2. Where can you create visible value fastest?
3. Where is evidence already strongest?
4. What wedge can you support operationally?

# Working logic

1. Compare segments by urgency, fit, and learnability.
2. Choose the wedge that increases speed of convergence.
3. Name why other wedges are not first.

# Output format

- recommended wedge
- 2 rejected wedges with reasons
- key risk to watch

# Review checklist

- Is the wedge narrow enough to act on?
- Is the rejection logic explicit?
- Does the recommendation increase learning speed?
