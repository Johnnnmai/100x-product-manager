# SKILL: organization-memory

## Overview
Capture, organize, and retrieve learnings across the organization.

## When to Use
- After experiments complete
- Post-launch retrospectives
- Building institutional knowledge
- Onboarding team members

## Inputs
- Raw learnings (from experiments, launches, etc.)
- Categorization scope
- Retrieval needs

## Process

### 1. Capture
- Extract key learnings from reports
- Document decisions and rationale
- Tag with relevant metadata

### 2. Organize
- Structure into knowledge categories
- Link related concepts
- Maintain relationships

### 3. Retrieve
- Enable semantic search
- Provide context-aware answers
- Suggest relevant past learnings

## Outputs
- Organized knowledge base
- Decision logs
- Lessons learned repository
- Retrieval API

## Knowledge Categories
- What worked (wins)
- What didn't (failures)
- Why things happened (causality)
- How to replicate (playbooks)
- What to avoid (gotchas)
