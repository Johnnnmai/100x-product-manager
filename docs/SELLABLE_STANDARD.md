# Sellable Standard for RoomJoy

This document defines what "good enough to sell" means.

RoomJoy does not need to be a breakout startup.
It needs to be a believable, functional, transferable micro-SaaS asset.

## Sellable Definition
RoomJoy is sellable when a buyer can reasonably believe:
1. the product works,
2. the product can charge,
3. the product can be maintained,
4. the product can be handed over without chaos.

## Sellability Dimensions

### 1. Product Completeness
Score 0 to 2

0 = mostly marketing page only
1 = some app flow exists, but core loop is incomplete
2 = user can sign up, upload, generate, view result, download, and access history

### 2. Monetization Readiness
Score 0 to 2

0 = no usable payment or credit logic
1 = pricing exists but backend monetization is partial
2 = paid plan and credit flow work end to end

### 3. Buyer Transferability
Score 0 to 2

0 = undocumented, hard to hand over
1 = partial docs and limited transfer readiness
2 = README, env docs, deploy docs, admin basics, and handover summary exist

### 4. UI Polish
Score 0 to 2

0 = looks unfinished or broken
1 = usable but rough
2 = clean, consistent, modern, and trustworthy enough for a buyer demo

### 5. Technical Stability
Score 0 to 2

0 = frequent failures, broken builds, severe errors
1 = partially stable with known issues
2 = key flows build and run reliably with no major console or runtime blockers

### 6. Sale Readiness
Score 0 to 2

0 = no sale packaging
1 = partial screenshots or partial materials
2 = demo assets, screenshots, positioning copy, and handover docs are ready

## Passing Threshold
Minimum sellable threshold = 9/12

Target threshold = 10/12 or higher

## Hard Gates
RoomJoy is NOT sellable if any of the following are missing:
- sign up/sign in
- upload flow
- generation result flow
- at least one paid checkout path
- basic docs
- no demo screenshots
- broken landing page
- broken core app routes

## What Not To Optimize Prematurely
Do not spend major time on:
- pixel perfection
- animations beyond basic polish
- SEO blog content
- broad style library
- advanced dashboards
- referral or viral loops

## Buyer Narrative
The buyer should be able to understand RoomJoy in under 3 minutes:

- what it does
- who it serves
- how users pay
- how to run it
- what to improve next

Everything built should support that narrative.
