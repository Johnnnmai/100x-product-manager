# 100X PM Codex Master Brief

## Canonical Identity

- Public brand: `100X PM`
- Target repo slug: `100x-pm-skills`
- Hero subtitle: `AI Product OS for PMs and Founders`

## Core Positioning

100X PM is not a prompt dump and not a generic PM template library.

It is an AI Product OS that helps PMs and founders:

- turn rough ideas into compelling, scoped, shippable directions
- validate whether demand is real
- write specs for both humans and agents
- define metrics, experiments, evals, fallback, and guardrails
- turn roadmaps into delegated human and agent execution
- optimize context, memory, and token usage

## Product Definition

100X PM should not be positioned as working harder or just writing faster.

Define it as:

- turning ambiguity into decisions
- moving through aggressive iteration
- shipping when execution cost is lower than thinking cost
- designing AI behavior through prompts, evals, fallback, and constraints
- using systems and leverage to create outsized outcomes

## Product Layers

### Direction

Answer:

- is this worth doing
- who is worth serving first
- what wedge to use
- how to scope down now

### Decision

Answer:

- what success means
- what trade-offs are real
- how the PRD should work for humans and agents
- how experiments and AI evals should be designed

### Delegation

Answer:

- how the roadmap is split
- what goes to agents vs humans
- how work is tracked
- how context is compressed and reused

## Methodology To Bake In

### Context Engineering

- smallest high-signal context
- explicit task contracts
- compression before context bloat
- separate active context, memory, and archive

### Skill Design

- progressive disclosure
- composability
- portability across Codex, Claude Code, and similar agents

### Ralph Loop Logic

- classify the task type
- define default iterations and max iterations
- define stop conditions
- define escalation conditions
- use cheaper models for bounded loops where appropriate

### Agent Operating Rules

- autonomous-first exploration for rough ideas
- enumerate meaningful options when ambiguity is high
- do not present simulated insight as validated truth
- capture reusable learnings after review or correction
