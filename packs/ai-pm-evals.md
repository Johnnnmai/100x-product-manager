# AI PM & Evals Pack

Specify AI behavior, define evals, contain failure, and raise product quality.

## Included skills

- `ai-prompt-spec-for-pm` — Turn a PM requirement into an AI behavior spec with inputs, outputs, and constraints.
- `multi-step-agent-spec` — Define a multi-step agent flow with state, transitions, and handoff rules.
- `eval-design-for-ai-feature` — Define how an AI feature should be tested, scored, and judged before launch.
- `eval-case-generator` — Generate normal, edge, and red-flag cases for evaluation.
- `eval-scorecard-builder` — Create a reusable scorecard for human or AI evaluators.
- `rubric-threshold-setter` — Set launch, review, and stop thresholds from a scoring rubric.
- `fallback-and-guardrail-designer` — Define fallback paths, blocked outputs, and handoff logic for risky AI behavior.
- `model-behavior-debugger` — Diagnose whether bad AI behavior comes from spec, prompt, eval, or fallback design.
- `ai-feature-readiness-check` — Check whether an AI feature is ready to ship across spec, eval, fallback, and monitoring.
